It might be better to divide each device capability into a separate API spec, like Battery Status API does. Putting so many API categories into one spec would make it hard to be reviewed and identify the real usage scenarios.
This specification defines a set of System Level APIs to allow Web applications retrieve the status of the device capabilities supported and system information from the underlying system.

This document is intended to define the Device Capabilities API to retrieve the status of hardware device, get the value of selected properties, and subscribe to asynchronous notifications of some specific properties in Web applications. To avoid pushing an over-complicated and overlapped API definitions, it firstly focuses on some general device capabilities, like CPU, memory, storage and display and so on. This specification is easy to be extended for supporting a new device capability during API evolution.

Note that this specification doesn't attempt to re-define device APIs which are already officially published by W3C working group. For example, Battery Status API , Network Information API , and Screen Orientation API . If the published device API specification cannot satisfy the requirements, the right way is to consider how to extend it.

Introduction

The Device Capabilities API provides web apps a way to retrieve device and system information from the underlying operating system. Device capabilities vary a great deal across the wide range of devices applications run on. This API lets applications take advantage of the capabilities of the device the application is running on. For example, knowing the number of CPU cores allows Web applications to determine an appropriate number of Web Workers to create for a worker pool for a compute intensive task. Knowing which sensors are supported, allows an application to know it can use these sensors to provide an alternative means of user input, perhaps by moving the device. It is often the case that other APIs let an application detect capabilities one by one through testing, but this is too slow for quickly determining the initial presentation to the user.

Security and Privacy Considerations

A Web application using this API is granted to access some device specific data which may contain information that is treated as private by end-user. For instance a user may become sensitive to a Web application retrieve the available storage free space. Therefore, a conforming implementation of this specification MUST provide a mechanism that protects the user's privacy and this mechanism should ensure that no information exposed by this API is retrievable without the permission granted.

DeviceCapabilities interface

The DeviceCapabilities interface is exposed on the Navigator object.

readonly attribute DeviceCapabilities system
The object that exposes the device capabilities retrieval functionality.

DeviceCapabilities interface encapsulates a set of methods to retrieve the hardware capabilities and specific device data, e.g. CPU, storage and display, and event handler to register asynchronous notification when the interested system events are triggered.

The methods defined in DeviceCapabilities are designed to return a Promise object and follow a specific pattern when invoked:

  1. Make a request to the system to retrieve the specified information implied by the method name;
  2. Let promise be a new Promise object and resolver its associated resolver.
  3. Return promise and run the remaining steps asynchronously.
  4. If an error occurs, run these substeps and then terminate these steps:
    1. Let error be a new DOMError object whose name is "UnknownError",
    2. Run resolver's internal reject algorithm with error as value.
  5. If the operation successfully completes, run these substeps:
    1. Let data be a new object for the specified device information (e.g. DeviceCPU, DeviceStorage and DeviceDisplay) that were retrieved,
    2. Run resolver's intenal fulfill algorithm with data as value.
Promise getCapabilities()
Retrieve the hardware capabilities supported by the underlying system on which the Web application is running.
Promise getCPUInfo()
Retrieve the specific information for CPU
Promise getMemoryInfo()
Retrieve the memory information
Promise getStorageInfo()
Retrieve the storage information
Promise getDisplayInfo()
Retrieve the display information

Storage

readonly attribute DeviceStorageUnit[] storages
An array of DdeviceStorageUnit objects
attribute EventHandler onattach
Fired when a new storage is attached to the system. The parameter passed to the event handler is the DeviceStorageUnit being connected to the system.
attribute EventHandler ondetach
Fired when a new storage is detached to the system. The parameter passed to the event handler is the DeviceStorageUnit being disconnected to the system.

DeviceStorageUnit interface

attribute readonly DOMString id
The unique identifier of the storage device
attribute readonly DOMString name
The human-readable name
attribute readonly DOMString type
The type of storage device, should be one of 'fixed', 'removable', 'unknown'
attribute readonly double capacity
The total capacity of the storage devices, in bytes

Display

readonly attribute DeviceDisplayUnit[] displays
An array of DeviceDisplayUnit objects
attribute EventHandler onconnect
Fired when a new display is connected to the system. The parameter passed to the event handler is a DeviceDisplayUnit object being connected to the system.
attribute EventHandler ondisconnect
Fired when a display is removed from the system. The parameter passed to the event handler is a DeviceDisplayUnit object being connected to the system.

DeviceDisplayUnit interface

attribute readonly DOMString id
The unique identifier of the display.
attribute readonly DOMString name
The user-friendly name (e.g. "HP LCD monitor").
attribute readonly boolean isPrimary

attribute readonly boolean isInternal
True if this is an internal display.
